# Product Video

## 核心目标

把用户直接上传或提供的商品图与参考图变成可继续制作视频的分镜与提示词。默认采用“极速模式”：识别到可用商品图后立即自动锁定商品、设计分镜并开始出图，不先提问或等待确认。

聊天附件默认先生成一张无叠字的五格 9:16 场景分镜图，让用户上传后立刻看到结果；用户要求“拆分分镜/单独出图”时，再按顺序生成独立关键帧。只有用户消息中显式给出本地文件或目录路径时，才默认生成完整文件素材包；附件内部临时路径不改变聊天模式。

只使用当前 Codex 会话提供的图片查看与内置图片生成/编辑能力。不要配置、读取、索取或调用外部识图、生图、视频或去水印 API。

## 必读引用

按任务阶段完整读取对应文件：

- 开始单图或项目任务前，读取 [references/workflow.md](references/workflow.md)。
- 组装商品增强、关键帧或重试 prompt 前，读取 [references/prompts.md](references/prompts.md)。
- 输出 Flow、CapCut 或 JSON 提示词前，读取 [references/platforms.md](references/platforms.md)。
- 做商品一致性检查、补跑或报告前，读取 [references/quality.md](references/quality.md)。

可用脚本：

- `scripts/preflight_product_video.py`：扫描输入、读取客观尺寸和基于路径的角色提示，建立 `source-manifest.json`。
- `scripts/validate_product_video.py`：校验素材包结构、JSON 字段、关键帧数量和宽高比，写入 `qc-report.md`。

## 默认规格

用户未指定时使用：

```text
平台：Google Flow + CapCut
画幅：竖屏 9:16
成片目标：约 10 秒
镜头数：5
镜头结构：开场吸引 / 商品主展示 / 使用场景 / 细节特写 / 结尾定格
提示词：中文操作说明 + 英文生成提示词
确认次数：0 次（极速模式）
```

用户明确指定平台、比例、时长或镜头数时，以用户要求覆盖对应默认值。

## 商品真值规则

把图片分为两种证据，禁止混用：

- **商品真值图**：用于锁定产品外形、颜色、数量、结构、Logo、包装文字和部件。优先使用清晰正面图，再使用侧面/背面/细节图补充。
- **风格参考图**：只用于提取场景、光线、色调、构图、道具和镜头语言。即使画面里有相似产品，也不能覆盖商品真值。

模糊风格图可以直接使用。模糊商品图先自动生成保守增强候选，并把看不清的文字、Logo、接口和结构列入 `uncertainties`；不要把生成候选冒充原始事实。

只在图片损坏、完全看不出商品轮廓，或同时上传多个明显不同商品且无法判断目标时停下来询问。其它不确定项保守处理，不阻止开始。

## 执行模式

默认使用 **极速模式**：

- 单张上传图默认视为主要商品图。
- 多张上传图自动选择最清晰、商品最完整的一张为主要真值，其余相同商品视角作为补充；明显偏场景、氛围或构图的图片作为风格参考。
- 没有风格图时，依据商品外观选择安全、通用的商业场景，不虚构产品功能。
- 用户没给创意重点时，默认只传达“一眼看清并记住这个商品”，不编造功能或卖点；单图且用途不明时自动选择商品揭晓/英雄展示方向。
- 为五个镜头锁定同一套色彩、灯光、场景世界和主要运镜语言；开头 2 秒内清楚出现商品，结尾至少稳定保持约 2 秒。
- 自动写入商品身份卡并把 `confirmation_status` 设为 `auto_locked`，随后立即出分镜图。
- 不展示中间身份卡，不等待“确认”。

只有用户明确说“先给我确认/审阅模式/不要直接生图”时，才改用 **审阅模式**：展示图片分类、商品身份卡、不确定项、增强候选和五镜头摘要，确认后再生图。

## 生图执行规则

- 本地图片先查看再编辑；聊天附件没有本地路径时，直接使用最近的会话图片，不要求用户另存或重新上传。
- 本地目标图全部有路径时使用 `referenced_image_paths`；附件没有路径时使用能覆盖全部目标附件的最小 `num_last_images_to_include`，不要同时使用两种引用方式。
- 极速模式先用一次调用生成一张五格分镜联系表；拆分独立关键帧或制作完整素材包时，每次只生成一个镜头，当前镜头保存或记录成功后再处理下一个。
- 每次携带已锁定的主要商品图和必要的补充视角；只加入与当前镜头有关的风格参考。
- 始终要求保持真实商品的外形、颜色、数量、比例、Logo、包装文字和结构。
- 默认不添加画面叠字、卖点、徽章或虚构包装文字；商品本身已有文字必须尽量保持。
- 画幅不合格时用内置图片编辑扩展画布后重试；禁止用本地拉伸或破坏性裁切伪造 9:16。
- 单镜头最多生成 3 次。仍不合格时记录到 `failed.txt`，继续后续镜头。
- 聊天附件极速模式直接在会话中展示五格分镜图，不创建文件素材包。用户需要完整素材包时，请其在消息中提供明确的本地图片/目录路径；不能落盘时不要用占位图冒充关键帧。

## 推荐执行顺序

1. 检测当前消息中的上传图片；有附件时立即查看并自动分类，不询问路径。
2. 普通聊天附件在内存中建立商品身份锁、单一创意重点和五镜头分镜，直接生成五格场景分镜图。
3. 用户消息显式提供本地路径时运行预检脚本，自动写入 `auto_locked` 商品身份卡并直接生成完整素材包。
4. 用户要求拆分时，按镜头顺序生成、保存和质检独立关键帧。
5. 写入或输出 Flow、CapCut 和通用提示词。
6. 有文件素材包时运行校验脚本，按失败项单镜头补跑。

## 输出目录

用户消息显式给出本地输入路径时，默认在输入项目同级创建：

```text
项目名-视频素材包/
  source-manifest.json
  product-profile.json
  creative-brief.md
  storyboard.json
  storyboard.md
  keyframes/
  prompts/
    flow.md
    capcut.md
    universal.json
  qc-report.md
  failed.txt          # 仅有失败时
```

已存在的合格结果默认复用，只补缺失或失败镜头。只有用户明确要求“重做/覆盖/全部重新生成”时才覆盖成功结果。

## 最终回复

文件素材包完成后汇报输出目录、自动锁定状态、目标规格、关键帧成功数、失败数、提示词和质检报告路径。纯聊天极速模式以生成的分镜图作为本轮结果，不在生图结果后追加说明；后续可响应“拆分第几个镜头”或在用户提供本地输入路径后生成完整素材包。
