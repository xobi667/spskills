# spskills

可直接安装到 Codex 的 Skills 集合。

## product-video

上传一张商品图就开始。Codex 用内置生图能力自动完成一次内部导演、建立连续镜头链，并输出可直接用于 Seedance、Google Flow、CapCut 等视频生成工具的首尾帧和提示词。

- 自动在原图旁新建 `<商品名>_导演台_时间` 项目文件夹。
- 先在内部比较创意方向，再执行产品忠实、因果清楚、可连续生成的一套。
- 5 段视频共用 6 张 9:16 锚点帧：`01→02`、`02→03`、`03→04`、`04→05`、`05→06`。
- 每段只有一个可见动作、一种运镜和一个明确落点；不再生成五张互不相关的商品海报。
- 强制检查重复机位、上下前后机械轮换、商品复制、完整倒影、动作重启和方向跳变。
- 最终目录只有 `01.png` 至 `06.png` 与一个 UTF-8 BOM `导演台.txt`；TXT 的 01–05 每段恰好对应一条可复制提示词。
- 不需要外部 API Key，不安装或运行本地图片/视频大模型。
- 自动检测本机 `lark-cli`，优先把图片与纯文本发给当前登录的飞书本人；飞书不可用时仍完整保存在本地。
- 安全运行日志默认写入桌面的 `生图日志.txt`，不记录 token、用户 ID 或聊天 ID。

### 安装

在 Codex 中运行：

```text
使用 $skill-installer 安装 https://github.com/xobi667/spskills/tree/main/product-video
```

也可以把 `product-video` 目录复制到：

```text
~/.codex/skills/product-video
```

安装或更新后新建一个 Codex 会话，让 Skill 列表重新加载。

### 使用

选择“商品视频导演台”，直接上传商品图并发送；也可以附图后输入：

```text
$product-video 直接开始
```

本地文件或目录也可以直接指定：

```text
$product-video D:\商品项目\主图.png
$product-video D:\商品项目
```

默认输出示例：

```text
主图_导演台_20260714-153000\
├─ 01.png
├─ 02.png
├─ 03.png
├─ 04.png
├─ 05.png
├─ 06.png
└─ 导演台.txt
```

`导演台.txt` 中的每段映射固定为：

```text
01：01.png → 02.png
02：02.png → 03.png
03：03.png → 04.png
04：04.png → 05.png
05：05.png → 06.png
```

如果当前 Codex 会话没有内置生图工具，Skill 不会用本地占位图冒充结果；它只会把简短失败原因和五条备用提示词写进同一个 TXT，提示词需补齐首尾锚点后使用。
