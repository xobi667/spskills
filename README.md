# spskills

可直接安装到 Codex 的 Skills 集合。

## product-video

上传商品图或指定商品目录就开始。Codex 先视觉核对同一 SKU，自动识别窗帘/窗纱、窗膜、贴纸、姓名贴、杯封膜、桌面覆盖和节庆装饰等路线，再按真实材料物理完成“生活需求→准备→操作→验证→回到生活”的五段闭环。结果可直接用于 Seedance、Google Flow、CapCut 等视频生成工具。

- 自动在原图旁新建 `<商品名>_导演台_时间` 项目文件夹。
- 不只看文件夹名：先视觉核对同款颜色、图案、文字、边框、卷芯、配件和安装方式，避免混目录、跨款和下游 AI 合成图污染。
- 建立身份、比例、安装、动作、结果和精确印刷参考包；只有清晰结果证据可以授权功能主张。
- 按品类锁定合法形变和动作：窗帘服从重力、纱网保持张力、窗膜与玻璃共面、贴纸保持图案文字、封口膜保持卷材和机器走膜连续。
- 生图前用确定性校验器检查路线、形态、六个生活阶段、品类×阶段动作白名单、接触、速度/阻力、事实权限、自然主义和回到生活；失败先改导演表。
- 5 段视频共用 6 张 9:16 锚点帧：`01→02`、`02→03`、`03→04`、`04→05`、`05→06`。
- 每段只有一个真实动作和一种有动机的相机行为；01 先作为场景预览，合格后固定为全片 `world_plate`。
- 默认普通住宅、教室或小店，35–50mm 观感和单一自然/实用光；拒绝黑棚底座、漂浮、粒子、光束、火花、豪宅样板间和英雄海报。
- 最终目录只有 `01.png` 至 `06.png` 与一个 UTF-8 BOM `导演台.txt`；TXT 包含商品路线、日常场景、完整闭环、一个传播重点、五条可执行提示词和 0–3 句具体文案。
- 不需要外部 API Key，不安装或运行本地图片/视频大模型。
- 自动检测本机 `lark-cli`，优先把图片与纯文本发给当前登录的飞书本人；飞书不可用时仍完整保存在本地。
- 安全运行日志默认写入桌面的 `生图日志.txt`，不记录 token、用户 ID 或聊天 ID。

### 安装

在 Codex 中运行：

```text
使用 $skill-installer 安装 https://github.com/xobi667/spskills/tree/main/product-video
```

也可以把 `product-video` 目录复制到：

```text
~/.codex/skills/product-video
```

安装或更新后新建一个 Codex 会话，让 Skill 列表重新加载。

### 使用

选择“自然商品视频导演台”，直接上传商品图并发送；也可以附图后输入：

```text
$product-video 直接开始
```

本地文件或目录也可以直接指定：

```text
$product-video D:\商品项目\主图.png
$product-video D:\商品项目
```

默认输出示例：

```text
主图_导演台_20260714-153000\
├─ 01.png
├─ 02.png
├─ 03.png
├─ 04.png
├─ 05.png
├─ 06.png
└─ 导演台.txt
```

`导演台.txt` 的开头会明确商品路线、日常场景、完整闭环与传播重点，每段映射固定为：

```text
01：01.png → 02.png
02：02.png → 03.png
03：03.png → 04.png
04：04.png → 05.png
05：05.png → 06.png
```

每段还会给出生活动作、视频提示词和字幕/旁白。文案可以全部不用，最多三句；每句都要绑定具体物体、动作或有证据结果。“重新定义品质”“细节自会说话”“一步到位”等空话会被拒绝。

如果当前 Codex 会话没有内置生图工具，Skill 不会用本地占位图冒充结果；它只会把简短失败原因和五条备用提示词写进同一个 TXT，提示词需补齐首尾锚点后使用。
